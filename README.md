# Text Autocomplete: LSTM vs Transformers

Проект по разработке легковесной системы автодополнения текста для мобильного приложения социальной сети.

## 📌 Бизнес-контекст и задача

В современном соцсетевом приложении необходимо реализовать функцию автодополнения коротких постов. Главный вызов — **ограничения мобильных устройств** по оперативной памяти и скорости работы. Требуется создать модель, которая будет достаточно точной, но при этом максимально легковесной.

**Формальная задача:**

1. Подготовить и очистить датасет коротких постов.
2. Реализовать и обучить рекуррентную нейросеть (LSTM) "с нуля".
3. Сравнить её качество с относительно тяжелой предобученной моделью (`distilgpt2`).
4. Дать рекомендацию: стоит ли использовать легкую модель или ресурсы позволяют внедрить трансформер.

## 🛠 Стек технологий

* **Package Manager:** [uv](https://astral.sh)
* **Deep Learning:** PyTorch, Hugging Face Transformers (`distilgpt2`)
* **Data Science:** Pandas, Scikit-learn, PyYAML

## 🧠 Методология

### 1. Обработка данных

Изначально датасет — это набор текстов. Для обучения предсказанию следующего токена данные преобразуются в последовательности, где целью (target) является входной текст, смещенный на один токен влево для каждого шага предсказания.

### 2. Архитектура LSTM

Реализована RNN-модель на базе LSTM-слоев, поддерживающая два режима работы:

* **Обучение:** Оптимизация функции потерь **Cross-Entropy Loss** для предсказания распределения вероятностей следующего токена на каждом шаге последовательности.
* **Генерация:** Авторегрессионное предсказание (token-by-token) с использованием стратегии **Greedy Search** до достижения токена завершения или лимита длины.

### 3. Сравнение с SOTA

В качестве базовой линии (baseline) используется **distilgpt2** — дистиллированная версия GPT-2. Это модель архитектуры **Transformer (Decoder-only)**, оптимизированная по размеру, но сохраняющая высокую точность генерации.

## 📈 Метрики качества

Для оценки качества генерации коротких текстов используются:

* **ROUGE-1:** Совпадение одиночных слов (unigrams).
* **ROUGE-2:** Совпадение пар слов (bigrams) для проверки связности.
